# Version knowledge sources and promote only successful versions

知识资料采用版本化模型：内容完全相同的上传复用已有资料版本，内容变化产生不可变的新版本；只有新版本完成全部必要处理阶段后才成为当前版本。这样会增加版本关系与存储成本，但能够避免重复炼制，并确保新版本处理失败时不破坏已有可用知识。

## Considered Options

- 每次上传都创建独立知识资料：实现直接，但会产生无法辨认的重复知识。
- 上传后立即覆盖当前内容：节省存储，但失败或中断会破坏已有可用知识。
- 保留不可变资料版本并在成功后切换：采用此方案，以换取可追溯性和失败隔离。
