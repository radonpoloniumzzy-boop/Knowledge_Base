# Migrate existing knowledge incrementally in place

现有知识库采用备份后的增量迁移，不通过空数据库和重新扫描重建。现有可用资料映射为知识资料的首个资料版本，已有标准知识文档、分块、标签、衍生产物、能力包、提示词和反馈尽量保持原关联；无法可靠判断的关系进入迁移报告，不猜测合并，也不自动重新处理内容。

## Considered Options

- 创建空数据库并重新扫描：实现简单，但会丢失反馈、标签状态、能力包配方和历史关系。
- 直接修改正式数据库：速度快，但失败时恢复风险高。
- 在副本上演练增量迁移并验证后替换：采用此方案，以保护现有知识资产并提供回退路径。

## Consequences

- 数据库需要正式的 schema 版本和顺序迁移机制。
- 迁移前必须生成带时间戳的备份，并对关键记录数量进行前后核对。
- 旧的卡住或重复任务归档为历史，不在迁移时自动重新执行。
