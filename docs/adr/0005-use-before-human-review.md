# Make successful knowledge available before human review

核心处理和基础质量检查成功后，资料版本立即成为当前版本并标记为“可用·未复核”，不设置事前人工审批。低置信度和普通警告通过状态、证据及能力包阈值管理；只有质量阻断才使版本进入“需要处理”。用户在实际使用中发现的问题通过事后编辑和反馈记录修正。

## Considered Options

- 所有资料人工确认后才能使用：控制严格，但无法支撑大量历史资料和按需复核习惯。
- 自动开放所有处理结果：省事，但严重解析问题可能进入知识库。
- 基础质量检查后立即开放并标记未复核：采用此方案，在低人工负担与最低质量保障之间取平衡。

## Consequences

- 检索结果和资料详情需要显示复核状态与重要警告。
- 人工反馈是按需纠正机制，不是导入任务的必经阶段。
- 核心处理必须包含足以识别质量阻断的基础检查。
