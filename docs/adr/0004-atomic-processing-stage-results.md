# Commit processing stage results atomically

每个处理阶段采用完整提交语义。处理过程先产生临时结果，只有整个阶段成功后才一次性成为正式阶段结果；崩溃、超时或校验失败留下的中间内容不可被用户、检索或能力包使用，恢复时清理或覆盖中间内容并重新执行该阶段。

## Considered Options

- 处理过程中直接写入正式结果：可以实时看到进度，但崩溃后无法区分半成品与完整结果。
- 每个阶段完整提交：采用此方案，以换取可靠恢复和一致的可见性。

## Consequences

- 文件生成和批量数据库写入需要临时区域或等效的提交机制。
- 外部 AI 响应只有完整接收并通过校验后才能成为正式衍生产物。
- 进度显示来自导入任务状态，不来自半成品内容。
