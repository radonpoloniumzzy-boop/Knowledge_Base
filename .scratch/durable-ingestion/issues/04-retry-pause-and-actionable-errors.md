# 04 — 加入失败分类、自动重试和暂停继续

**What to build:** 导入任务能够区分临时失败与确定性失败，自动处理可恢复问题，并让用户安全暂停、继续和修正失败任务。

**Blocked by:** 03 — 完成可原子恢复的文本核心处理

**Status:** ready-for-agent

- [ ] 临时失败按递增等待时间自动重试，最多三次
- [ ] 重试次数和下次重试时间在应用重启后保持不变
- [ ] 确定性失败立即进入需要处理，不进行无意义自动重试
- [ ] 三次临时失败后任务进入需要处理，并保留失败阶段与可读原因
- [ ] 排队任务可以立即暂停，活动任务在当前阶段安全完成后暂停
- [ ] 继续任务从第一个未完成或失败阶段推进
- [ ] 页面不显示原始框架异常或 Python 异常载荷
