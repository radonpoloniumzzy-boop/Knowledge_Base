# 01 — 建立安全迁移与测试基础

**What to build:** 建立可重复运行的测试环境和数据库迁移机制，使后续资料导入改造可以先在隔离副本中验证，并在失败时恢复，不触碰现有知识资产。

**Blocked by:** None — can start immediately

**Status:** completed

- [x] 自动化测试可以使用临时 SQLite 数据库和临时受管理目录，不读写正式知识资产
- [x] 数据库记录明确的 schema 版本，并按顺序执行尚未应用的迁移
- [x] 迁移前生成带时间戳的数据库备份
- [x] 迁移可以在数据库副本上演练，并输出迁移前后关键记录数量
- [x] 数量核对失败或迁移异常时，正式数据库保持不变并可从备份恢复
- [x] 重复运行已经成功的迁移不会重复修改数据
- [x] 现有应用在扩展 schema 后仍可启动并使用旧导入流程

## Comments

- 正式数据库已从 schema 版本 0 迁移到版本 1；迁移前备份保存在受管理备份目录。
- 真实数据库副本演练中，文件、分块、标签关系、衍生产物、任务、能力包、提示词、设置和反馈数量迁移前后一致。
- 自动化测试 4 项通过，核心代码编译通过，仪表盘、资料导入、知识库、能力包和设置页面均返回 200。
- 项目当前不是 Git 仓库，因此本任务未创建 commit。
