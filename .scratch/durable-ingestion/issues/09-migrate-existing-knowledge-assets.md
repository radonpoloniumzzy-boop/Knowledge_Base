# 09 — 迁移现有知识资产

**What to build:** 将现有数据库和受管理知识安全映射到知识资料与资料版本模型，保留可靠关系，对不确定关系只报告不猜测。

**Blocked by:** 05 — 加入内容指纹和资料版本；07 — 拆出非阻塞知识增强；08 — 加入知识资料回收站

**Status:** ready-for-agent

- [ ] 正式迁移前自动生成带时间戳的数据库备份
- [ ] 迁移先在数据库副本中演练并输出关键数量核对
- [ ] 现有可用资料映射为首个资料版本，不自动重新处理内容
- [ ] 可靠的分块、标签、衍生产物、提示词、反馈、能力包和导出历史保持关联
- [ ] 无法可靠判断的历史关系进入迁移报告，不进行猜测合并
- [ ] 旧的卡住或重复任务归档为历史，不自动执行
- [ ] 数量核对或完整性验证失败时不替换正式数据库
- [ ] 迁移中断可以恢复备份并继续使用原应用数据
- [ ] 成功迁移后重复运行不会重复创建版本或关系
