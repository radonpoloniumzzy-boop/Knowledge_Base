# 09 — 迁移现有知识资产

**What to build:** 将现有数据库和受管理知识安全映射到知识资料与资料版本模型，保留可靠关系，对不确定关系只报告不猜测。

**Blocked by:** 05 — 加入内容指纹和资料版本；07 — 拆出非阻塞知识增强；08 — 加入知识资料回收站

**Status:** done

- [x] 正式迁移前自动生成带时间戳的数据库备份
- [x] 迁移先在数据库副本中演练并输出关键数量核对
- [x] 现有可用资料映射为首个资料版本，不自动重新处理内容
- [x] 可靠的分块、标签、衍生产物、提示词、反馈、能力包和导出历史保持关联
- [x] 无法可靠判断的历史关系进入迁移报告，不进行猜测合并
- [x] 旧的卡住或重复任务归档为历史，不自动执行
- [x] 数量核对或完整性验证失败时不替换正式数据库
- [x] 迁移中断可以恢复备份并继续使用原应用数据
- [x] 成功迁移后重复运行不会重复创建版本或关系

## Delivery

- 正式库升级到 schema version 9，并生成 schema 备份、旧资料迁移备份和 JSON 迁移报告。
- 5063 项可用旧知识资产映射为知识资料首版本；原文件、产物、分块、标签、能力包、提示词和反馈数量不变。
- 12 个旧上传任务归档；20252 项增强历史均为已完成或需要处理，没有自动排队和重新炼制。
- 144 项歧义写入报告：142 项同名资料保持独立，2 项旧反馈目标缺失。
