# 02 — 让文本资料进入持久任务队列

**What to build:** 用户上传 TXT 或 Markdown 后，每份资料形成可见的持久导入任务；任务按顺序单独推进，关闭并重启应用后仍存在并继续处理。

**Blocked by:** 01 — 建立安全迁移与测试基础

**Status:** done

- [x] 单个和批量文本上传都会为每份资料建立独立导入任务
- [x] 同一时间最多一个任务处于正在处理状态，其余任务保持等待
- [x] 任务状态、当前处理阶段和更新时间保存在持久存储中
- [x] 应用重启后等待和处理中任务仍能被任务队列发现
- [x] 重启时不会并行启动两个任务执行者
- [x] 资料导入页面显示等待、处理中、完成和需要处理的真实数量
- [x] FastAPI 与自动化测试通过同一个资料导入 interface 操作任务
