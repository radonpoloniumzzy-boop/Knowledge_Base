# 03 — 完成可原子恢复的文本核心处理

**What to build:** TXT 和 Markdown 资料经过内容提取、标准知识文档、基础质量检查、检索分块和当前版本切换，形成一条可恢复且半成品不可见的核心处理路径。

**Blocked by:** 02 — 让文本资料进入持久任务队列

**Status:** ready-for-agent

- [ ] 核心处理由有序且可独立确认完成的处理阶段组成
- [ ] 每个阶段结果完整提交，执行中断不会暴露半份文档或部分分块
- [ ] 重启后从第一个未完成阶段继续，已提交阶段不重复产生结果
- [ ] 标准知识文档和分块通过完整性验证后才可供检索
- [ ] 核心处理成功后资料版本成为当前版本并标记为可用·未复核
- [ ] 新版本切换失败时原当前版本保持可用
- [ ] 重复推进同一处理阶段不会产生重复正式记录或文件
